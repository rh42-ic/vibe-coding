# Verible Skill: SystemVerilog Workflow Guide

Verible is a suite of SystemVerilog developer tools. When developing SystemVerilog/Verilog code, it is recommended to follow this workflow to ensure code quality and project completeness.

## Recommended 3-Step Workflow

### Step 1: Code Generation

- Write or modify your `.v` or `.sv` files to implement the required logic. Ensure the changes are written to disk.

### Step 2: Format and Lint (Single-File Check)

- After saving, format and lint the modified files to catch basic syntax or style issues early. Fix any obvious errors before moving on.
- **Format:**
  ```bash
  verible-verilog-format --inplace <file.sv>
  ```
- **Lint:**
  ```bash
  verible-verilog-lint --rules_config_search <file.sv>
  ```
  _(The `--rules_config_search` flag ensures the linter searches upwards for the `.rules.verible_lint` project configuration file)._

### Step 3: Project Dependencies & Filelist

- Check if any instantiated sub-modules are missing their definition files. You can analyze the code directly or use `verible-verilog-project`.
- **Project:**
  ```bash
  verible-verilog-project file-deps --file_list_path filelist.f
  ```

---

## Configuration Wizards

Interactive terminal UIs to generate configuration files (run from the skill directory or use absolute paths):

- **Lint Rules:** `uv run scripts/generate_lint_rules.py` (Outputs `.rules.verible_lint`)
- **Format Style:** `uv run scripts/generate_format_config.py` (Outputs `.verible_format`)
