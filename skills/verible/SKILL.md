# Verible Skill (Quick Start)

Verible is a suite of SystemVerilog developer tools, including a linter, formatter, and language server.

## Tool Overview

| Tool | Purpose | Reference File |
|------|---------|----------------|
| `verible-verilog-lint` | Identifies coding style issues and bugs. | [verible-verilog-lint.md](references/verible-verilog-lint.md) |
| `verible-verilog-format` | Auto-formats code to a consistent style. | [verible-verilog-format.md](references/verible-verilog-format.md) |
| `verible-verilog-syntax` | Validates syntax and exports AST. | [verible-verilog-syntax.md](references/verible-verilog-syntax.md) |
| `verible-verilog-ls` | Language Server Protocol (LSP) for editors. | [verible-verilog-ls.md](references/verible-verilog-ls.md) |
| `verible-verilog-diff` | Syntax-aware diffing tool. | [verible-verilog-diff.md](references/verible-verilog-diff.md) |
| `verible-verilog-preprocessor` | Preprocessing and variant generation. | [verible-verilog-preprocessor.md](references/verible-verilog-preprocessor.md) |
| `verible-verilog-obfuscate` | Mangles identifiers while preserving structure. | [verible-verilog-obfuscate.md](references/verible-verilog-obfuscate.md) |
| `verible-patch-tool` | Applies patches generated by the linter. | [verible-patch-tool.md](references/verible-patch-tool.md) |
| `verible-verilog-project` | Manages file lists and dependencies. | [verible-verilog-project.md](references/verible-verilog-project.md) |

## Configuration Wizards

Interactive terminal UIs with RTL examples to help you generate configuration files.

| Wizard | Command | Output File |
|--------|---------|-------------|
| **Lint Rules Wizard** | `uv run scripts/generate_lint_rules.py` | `.rules.verible_lint` |
| **Format Style Wizard** | `uv run scripts/generate_format_config.py` | `.verible_format` |

*(Note: Run these from the skill's root directory or provide the full path to the scripts)*

## Core Task Recipes

### 1. Linting
Run the linter on one or more files:
```bash
verible-verilog-lint file.sv
```
Use `--rules_config_search` to find `.rules.verible_lint` automatically.
See [Lint Rules Reference](references/lint-rules.md) for a full list of rules.

### 2. Formatting
Format a file in-place:
```bash
verible-verilog-format --inplace file.sv
```
See [Formatting Flags Reference](references/formatting-flags.md) for styling overrides (indentation, column limits, etc.).

### 3. Syntax Checking
Check for syntax errors:
```bash
verible-verilog-syntax file.sv
```

### 4. Language Server
Start the LSP server (usually for IDE integration):
```bash
verible-verilog-ls
```
