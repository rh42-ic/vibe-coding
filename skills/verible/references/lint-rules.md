# Verible Lint Rules Reference

| Rule Name | Description | Enabled by default | Parameters |
|-----------|-------------|--------------------|------------|
| `always-comb` | Checks that there are no occurrences of `always @*`. Use `always_comb` instead. | yes | None |
| `always-comb-blocking` | Checks that there are no occurrences of non-blocking assignment in combinational logic. | yes | None |
| `always-ff-non-blocking` | Checks that blocking assignments are, at most, targeting locals in sequential logic. | yes | * `catch_modifying_assignments` Default: `false`<br>* `waive_for_locals` Default: `false` |
| `banned-declared-name-patterns` | Checks for banned declared name against set of unwanted patterns. | no | None |
| `case-missing-default` | Checks that a default case-item is always defined unless the case statement has the `unique` qualifier. | yes | None |
| `constraint-name-style` | Check that constraint names follow the required name style specified by a regular expression. | yes | * `pattern` Default: `([a-z0-9]+_)+c` |
| `create-object-name-match` | Checks that the 'name' argument of `type_id::create()` matches the name of the variable to which it is assigned. | yes | None |
| `dff-name-style` | Checks that D Flip-Flops use appropiate naming conventions in both input and output ports. The left hand side (output) and right hand side (input) are checked against a set of valid suffixes. Additionally, register names might end in a number to denote the pipeline stage index (var_q/var_q1, var_q2, ...). Pipelined signals must get their value from the previous stage: var_q3 <= var_q2. Exceptions to this rule can be configured using a regular expression or waiving whole `if` blocks | no | * `input` Default: `next,n,d` Comma separated list of allowed suffixes for the input port. Suffixes should not include the preceding "_". Empty field means no checks for the input port<br>* `output` Default: `reg,r,ff,q` Comma separated list of allowed suffixes for the output port. Should not include the preceding "_". Empty field means no checks for the output port<br>* `waive_ifs_with_conditions` Default: `!rst_ni,flush_i,!rst_ni \|\| flush_i,flush_i \|\| !rst_ni` Comma separated list of conditions that will disable the rule inside the `if`s they are evaluated in<br>* `waive_lhs_regex` Default: `(?i)mem.*` Nonblocking assigments whose lhs match the regex will not be evaluated |
| `disable-statement` | Checks that there are no occurrences of `disable some_label` if label is referring to a fork or other none sequential block label. Use `disable fork` instead. | no | None |
| `endif-comment` | Checks that a Verilog `` `endif`` directive is followed by a comment that matches the name of the opening `` `ifdef`` or `` `ifndef``. | no | None |
| `enum-name-style` | Checks that enum type names follow a naming convention defined by a RE2 regular expression. The default regex pattern expects "lower_snake_case" with either a "_t" or "_e" suffix. Refer to https://github.com/chipsalliance/verible/tree/master/verilog/tools/lint#readme for more detail on verible regex patterns. | yes | * `style_regex` Default: `[a-z_0-9]+(_t\|_e)` A regex used to check enum type name style. |
| `explicit-begin` | Checks that a Verilog ``begin`` directive follows all if, else, always, always_comb, always_latch, always_ff, for, forever, foreach, while and initial statements. | no | * `if_enable` Default: `true` All if statements require an explicit begin-end block<br>* `else_enable` Default: `true` All else statements require an explicit begin-end block<br>* `always_enable` Default: `true` All always statements require an explicit begin-end block<br>* `always_comb_enable` Default: `true` All always_comb statements require an explicit begin-end block<br>* `always_latch_enable` Default: `true` All always_latch statements require an explicit begin-end block<br>* `always_ff_enable` Default: `true` All always_ff statements require an explicit begin-end block<br>* `for_enable` Default: `true` All for statements require an explicit begin-end block<br>* `forever_enable` Default: `true` All forever statements require an explicit begin-end block<br>* `foreach_enable` Default: `true` All foreach statements require an explicit begin-end block<br>* `while_enable` Default: `true` All while statements require an explicit begin-end block<br>* `initial_enable` Default: `true` All initial statements require an explicit begin-end block |
| `explicit-function-lifetime` | Checks that every function declared outside of a class is declared with an explicit lifetime (static or automatic). | yes | None |
| `explicit-function-task-parameter-type` | Checks that every function and task parameter is declared with an explicit storage type. | yes | None |
| `explicit-parameter-storage-type` | Checks that every `parameter` and `localparam` is declared with an explicit storage type. | yes | * `exempt_type` Default: `` Set to `string` to exempt string types |
| `explicit-task-lifetime` | Checks that every task declared outside of a class is declared with an explicit lifetime (static or automatic). | yes | None |
| `forbid-consecutive-null-statements` | Checks that there are no occurrences of consecutive null statements like `;;` | yes | None |
| `forbid-defparam` | Do not use defparam. | yes | None |
| `forbid-line-continuations` | Checks that there are no occurrences of `\` when breaking the string literal line. Use concatenation operator with braces instead. | yes | None |
| `forbid-negative-array-dim` | Check for negative constant literals inside array dimensions. | no | None |
| `forbidden-macro` | Checks that no forbidden macro calls are used. | yes | None |
| `generate-label` | Checks that every generate block statement is labeled. | yes | None |
| `generate-label-prefix` | Checks that every generate block label starts with g_ or gen_. | yes | None |
| `instance-shadowing` | Warns if there are multiple declarations in the same scope that shadow each other with the same name. | no | None |
| `interface-name-style` | Checks that 'interface' names follow a naming convention defined by a RE2 regular expression. The default regex pattern expects "lower_snake_case" with a "_if" or "_e" suffix. Refer to https://github.com/chipsalliance/verible/tree/master/verilog/tools/lint#readme for more detail on regex patterns. | yes | * `style_regex` Default: `[a-z_0-9]+(_if)` A regex used to check interface name style. |
| `invalid-system-task-function` | Checks that no forbidden system tasks or functions are used. These consist of the following functions: `$psprintf`, `$random`, and `$dist_*`. As well as non-LRM function `$srandom`. | yes | None |
| `legacy-generate-region` | Checks that there are no generate regions. | no | None |
| `legacy-genvar-declaration` | Checks that there are no separate `genvar` declarations. | no | None |
| `line-length` | Checks that all lines do not exceed the maximum allowed length. | yes | * `length` Default: `100` Desired line length |
| `macro-name-style` | Checks that macro names conform to a naming convention defined by a RE2 regular expression. The default regex pattern expects "UPPER_SNAKE_CASE". Exceptions are made for UVM like macros, where macros named 'uvm_*' and 'UVM_*' follow "lower_snake_case" and "UPPER_SNAKE_CASE" naming conventions respectively. Refer to https://github.com/chipsalliance/verible/tree/master/verilog/tools/lint#readme for more detail on verible regex patterns. | yes | * `style_regex` Default: `[A-Z_0-9]+` A regex used to check macro names style. |
| `macro-string-concatenation` | Concatenation will not be evaluated here. Use `"...`" instead. | no | None |
| `mismatched-labels` | Check for matching begin/end labels. | no | None |
| `module-begin-block` | Checks that there are no begin-end blocks declared at the module level. | yes | None |
| `module-filename` | If a module is declared, checks that at least one module matches the first dot-delimited component of the file name. Depending on configuration, it is also allowed to replace underscore with dashes in filenames. | yes | * `allow-dash-for-underscore` Default: `false` Allow dashes in the filename where there are dashes in the module name |
| `module-parameter` | Checks that module instantiations with more than one parameter are passed in as named parameters, rather than positional parameters. | yes | None |
| `module-port` | Checks that module instantiations with more than one port are passed in as named ports, rather than positional ports. | yes | None |
| `no-tabs` | Checks that no tabs are used. Spaces should be used instead of tabs. | yes | None |
| `no-trailing-spaces` | Checks that there are no trailing spaces on any lines. | yes | None |
| `numeric-format-string-style` | Checks that string literals with numeric format specifiers have proper prefixes for hex and bin values and no prefixes for decimal values. | no | None |
| `one-module-per-file` | Checks that at most one module is declared per file. | no | None |
| `package-filename` | Checks that the package name matches the filename. Depending on configuration, it is also allowed to replace underscore with dashes in filenames. | yes | * `allow-dash-for-underscore` Default: `false` Allow dashes in the filename corresponding to the underscores in the package |
| `packed-dimensions-range-ordering` | Checks that packed dimension ranges are declare in little-endian (decreasing) order, e.g. `[N-1:0]`. | yes | None |
| `parameter-name-style` | Checks that parameter and localparm names conform to a naming convention based on a choice of 'CamelCase', 'ALL_CAPS' and a user defined regex ORed together. Empty configurtaion: no style enforcement. Refer to https://github.com/chipsalliance/verible/tree/master/verilog/tools/lint#readme for more detail on verible regex patterns. | yes | * `localparam_style` Default: `CamelCase` Style of localparam names<br>* `parameter_style` Default: `CamelCase\|ALL_CAPS` Style of parameter names.<br>* `localparam_style_regex` Default: `` A regex used to check localparam name style.<br>* `parameter_style_regex` Default: `` A regex used to check parameter name style. |
| `parameter-type-name-style` | Checks that parameter type names follow the lower_snake_case naming convention and end with _t. | no | None |
| `plusarg-assignment` | Checks that plusargs are always assigned a value, by ensuring that plusargs are never accessed using the `$test$plusargs` system task. | yes | None |
| `port-name-suffix` | Check that port names end with _i for inputs, _o for outputs and _io for inouts. Alternatively, for active-low signals use _n[io], for differential pairs use _n[io] and _p[io]. | no | None |
| `positive-meaning-parameter-name` | Checks that no parameter name starts with 'disable', using positive naming (starting with 'enable') is recommended. | yes | None |
| `posix-eof` | Checks that the file ends with a newline. | yes | None |
| `proper-parameter-declaration` | Checks that every `parameter` declaration is inside a formal parameter list of modules/classes and every `localparam` declaration is inside a module, class or package. | no | * `package_allow_parameter` Default: `false` Allow parameters in packages (treated as a synonym for localparam).<br>* `package_allow_localparam` Default: `true` Allow localparams in packages. |
| `signal-name-style` | Checks that signal names conform to a naming convention defined by a RE2 regular expression. Signals are defined as "a net, variable, or port within a SystemVerilog design". The default regex pattern expects "lower_snake_case". Refer to https://github.com/chipsalliance/verible/tree/master/verilog/tools/lint#readme for more detail on verible regex patterns. | no | * `style_regex` Default: `[a-z_0-9]+` A regex used to check signal names style. |
| `struct-union-name-style` | Checks that `struct` and `union` names use lower_snake_case naming convention and end with '_t'. | yes | * `exceptions` Default: `` Comma separated list of allowed upper-case elements, such as unit-names |
| `suggest-parentheses` | Recommend extra parentheses around subexpressions where it helps readability. | yes | None |
| `suspicious-semicolon` | Checks that there are no suspicious semicolons that might affect code behaviour but escape quick visual inspection | no | None |
| `truncated-numeric-literal` | Checks that numeric literals are not longer than their stated bit-width to avoid undesired accidental truncation. | yes | None |
| `typedef-enums` | Checks that a Verilog `enum` declaration is named using `typedef`. | yes | None |
| `typedef-structs-unions` | Checks that a Verilog `struct` or `union` declaration is named using `typedef`. | yes | * `allow_anonymous_nested` Default: `false` Allow nested structs/unions to be anonymous. |
| `undersized-binary-literal` | Checks that the digits of binary literals for the configured bases match their declared width, i.e. has enough padding prefix zeros. | yes | * `bin` Default: `true` Checking binary 'b literals.<br>* `oct` Default: `false` Checking octal 'o literals.<br>* `hex` Default: `false` Checking hexadecimal 'h literals.<br>* `lint_zero` Default: `false` Also generate a lint warning for value zero such as `32'h0`; autofix suggestions would be to zero-expand or untype `'0`.<br>* `autofix` Default: `true` Provide autofix suggestions, e.g. 32'hAB provides suggested fix 32'h000000AB. |
| `unpacked-dimensions-range-ordering` | Checks that unpacked dimension ranges are declared in big-endian order `[0:N-1]`, and when an unpacked dimension range is zero-based `[0:N-1]`, the size is declared as `[N]` instead. | yes | None |
| `uvm-macro-semicolon` | Checks that no `uvm_* macro calls end with ';'. | no | None |
| `v2001-generate-begin` | Checks that there are no generate-begin blocks inside a generate region. | yes | None |
| `void-cast` | Checks that void casts do not contain certain function/method calls. | yes | None |
